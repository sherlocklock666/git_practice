<文件的4种状态>
Untracked：	未跟踪。在文件夹中，但没有加入git库，不参与版本控制。通过“git add”状态变为“Staged”。

Staged：	暂存状态。通过“git commit”将修改同步到库中，此时库中文件与本地文件有变为一致，文件为“Unmodify”状态。通过“git reset HEAD filename”取消暂存，文件状态为“Modified”。

（以下是对本地文件修改内容后的状态及操作）
Unmodify：	文件已入库，未修改。版本库中文件快照内容与文件夹中完全一致。如果被修改，变为“Modified”。通过“git rm”移出版本库，变为“Untracked”。

Modified：	文件已修改，仅仅修改，无其他操作。通过“git add”变为“Staged”状态，通过“git checkout”丢弃修改，返回“Unmodify”状态。“git checkout”即从库中取出文件，覆盖当前修改。

<查看文件状态>
git status：	查看所有文件状态
git status (filename)：	查看指定文件状态
git log：	查看提交历史（git log操作后想退出，在英文输入法状态下按q即可，测试能否还原）


<提交命令>
git add .：	添加所有文件到暂存区
git commit -m"自己输入提交信息"：	提交暂存区的文件到本地仓库

<分支管理>
git branch -a:	查看所有分支，红色是远程，带*号是当前
git branch -vv:	查看本地分支与远程分支对应关系
git checkout 分支名：	切换到该分支
